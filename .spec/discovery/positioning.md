# ポジショニング

## 競合代替

| 区分 | 代替 | 比較 |
|---|---|---|
| 現状維持 | 各リポジトリのAGENTS.mdとプロンプトを都度作る | 軽量だが更新・再利用・機械検証が分散する |
| プラットフォーム別 | Claude Code / Antigravity / Codex の設定を別々に整備 | ネイティブ機能を使えるが、同じ規律が3系統に分岐する |
| 汎用テンプレート | 一般的な開発規約やGitテンプレートをコピー | 汎用性は高いが、BitzLabs固有のSDD・権限分離・検証義務を持たない |
| 都度指示 | AIに毎回「安全に」「テストして」と依頼 | 導入不要だが再現性、追跡可能性、強制力が弱い |

## Points of Parity

- Git、テスト、ドキュメント、プラグイン導入の一般的な開発作法を案内できる。
- Agent Skills を再利用可能なフォルダ単位で配布できる。
- 各プラットフォームのプラグイン／マーケットプレイス機構を利用できる。

## Points of Difference

1. **BitzLabsの開発規律を実行可能な形で配布する** — 文書だけでなく、スキル、フック、スクリプト、
   マニフェスト、検証を一体化する。
2. **仕様・実装・検証を追跡する** — BitzSDDのEARS要件、タスク境界、検証証跡、権限分離を中核に置く。
3. **3プラットフォームで中核規律を共有する** — Agent Skills を共通層とし、仕様差をプラグイン境界で扱う。
4. **標準環境自身をドッグフーディングする** — BitzSkillsの変更にも同じブランチ、SDD、検証、リリース規律を適用する。
5. **安全な失敗を設計する** — 危険操作の禁止、worktreeによる分離、承認境界を標準に含める。

## ポジショニング・ステートメント

AIエージェントと開発する **BitzLabs の開発者**にとって、**BitzSkills** は、
ガードレール、Gitフロー、仕様駆動開発、ドメイン設計、拡張ツールを用途別プロファイルで提供する
**標準エージェント開発環境**である。プロジェクトごとの手書き規約やプラットフォーム別設定と異なり、
同じ中核規律をバージョン付きプラグインとして3環境へ配布し、機械検証可能な証跡まで含める。

## 市場カテゴリ

「組織内AIエージェント開発標準のプラグイン・ディストリビューション」。
外部市場の規模や直接競合は調査していないため `TBD`。

## 適合しない用途

- 完全なDeveloper Experienceプラットフォームや端末管理を求める用途。
- BitzLabsのSDD・Git・安全規律を採用しない組織への無変更導入。
- 1つのAIプラットフォーム固有機能を最大化することだけが目的の環境。
