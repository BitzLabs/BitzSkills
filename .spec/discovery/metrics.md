# 成功指標

> テレメトリと利用台帳は未整備。根拠のない目標値は置かず、現時点の数値は `TBD` とする。

## North Star Metric

**BitzSkills の標準プロファイルを利用し、定められた検証を通過して完了した開発サイクル数。**

インストール数やスキル起動回数ではなく、「共通規律が実際の開発を最後まで支えた回数」を測る。

| I1 | 標準環境適用プロジェクト数 | 標準プロファイルと共通ガードレールを導入したリポジトリ数 | BitzSkills 自身で運用 / 目標 `TBD` |

| I4 | 規約再説明の削減 | セッション内で同じガードレール・Git・SDD規律を再指示した回数 | `TBD` |
| I5 | 更新追従率 | 最新互換版へ更新され、doctor / release check を通過した導入環境の割合 | `TBD` |

## ガードレール指標

- **G1: 禁止操作違反 0件** — `rm -rf`、force push、hard reset、clean、sudo を実行しない。
- **G2: 配布ドリフト 0件** — 3マニフェストの name / version と marketplace / 実体を一致させる。
- **G3: 仕様の幽霊参照・孤児要件 0件** — 正規の workspace 一括検証で維持する。
- **G4: プラットフォーム固有情報の誤記 0件** — 各CLI実測または一次資料で検証する。
- **G5: ローカル変更の無断上書き 0件** — 更新・配置前に由来とバージョンを確認する。

## 計測計画

1. 当面は PR 本文の検証結果と `.spec/reports/` を手集計する。
2. 「標準環境導入済み」の機械判定方法を別要件として検討する。
3. 3件以上の適用プロジェクトまたは複数開発者の利用が得られた時点で目標値を設定する。
