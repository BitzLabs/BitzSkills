# ビジョン

**BitzLabs の誰がどの対応AIエージェントを使っても、安全性・仕様・Git・品質の基準が揃い、
プロジェクト固有の価値に集中できる開発状態をつくる。**

## ミッション

BitzSkills は、**BitzLabs でAIエージェントを用いて開発する際の標準作業環境**を、
Agent Skills オープン標準に準拠したクロスプラットフォームなプラグインとして
定義・検証・配布する。

## Vision Board

### Target Group

- 主要: BitzLabs のリポジトリをAIエージェントと開発・保守する開発者。
- served: その開発者の指示で実装・設計・検証を行う Claude Code、Antigravity 2.0、Codex。
- 将来: BitzLabs の開発規律を共有して参加する共同開発者。実利用は `TBD`。

### Needs

- プロジェクトごと、セッションごと、エージェントごとに開発規律を説明し直したくない。
- 危険操作、仕様と実装の乖離、Git運用のばらつきを、注意力ではなく仕組みで抑えたい。
- 利用ツールを変更しても、仕様・設計・検証・レビューの基準を維持したい。
- 標準環境自体をバージョン管理し、安全に更新・改善したい。

### Product

- `bitz-env`: ガードレール、環境診断、協調運用。機械フックは現在 Claude Code / Antigravity 対応で、
  Codexは収録スキルとAGENTS規律を利用する（機能差は SI-ENV-023 で追跡）。
- `bitz-sdd`: 仕様駆動の探索・設計・実装・検証・報告。
- `bitz-ddd`: ドメイン設計と既存コード評価。
- `bitz-flow`: SDD 非採用時にも使える Git / GitHub フロー。
- `skill-creator` / `plugin-creator`: 標準環境を拡張・検証・配布する開発基盤。
- 共有マーケットプレイス、3マニフェスト、リリース検査による配布整合。

### Business Goals

- 開発開始時の環境説明・再設定・判断の重複を減らす。
- リポジトリ間・エージェント間の規約ドリフトを検出可能にする。
- 安全性とトレーサビリティを保ったまま、エージェントによる開発速度を上げる。
- 数値目標は計測基盤がないため `TBD`。成功指標は CORE-DSC-002 で定義する。

## Values

1. **安全側に倒す** — 破壊的操作と権限逸脱をガードレールで防ぐ。
2. **仕様を正にする** — 意図・契約・実装・検証の追跡可能性を保つ。
3. **中核をポータブルにする** — Agent Skills を共通層とし、プラットフォーム差は境界に隔離する。
4. **機械で検証する** — 自己申告ではなくスクリプトとテストの実出力を根拠にする。
5. **小さく戻せる** — 1 PR = 1 関心事、featureブランチ、squash mergeを守る。

## PR-FAQ（圧力試験）

### プレスリリース要約

**「BitzSkills — BitzLabs の標準エージェント開発環境」**。
BitzLabs の開発者は、利用するAIエージェントにかかわらず、共通のガードレール、Git運用、
仕様駆動開発、ドメイン設計、品質検証をプラグインとして導入できる。標準は単一の巨大ツールではなく、
用途別プロファイルで必要なプラグインを組み合わせ、同じ規律と検証方法を共有する。

### FAQ

- **すべてのプラグインを必ず入れるのか?** いいえ。標準プロファイルを基準に、DDDや開発ツールを拡張する。
- **OSや言語SDKもセットアップするのか?** いいえ。現在の標準対象はAIエージェントの作業規律とプラグイン層。
- **1つのエージェント専用か?** いいえ。Claude Code、Antigravity 2.0、Codex CLIを配布対象とする。
  ただし配布可能性と機能同等性は分けて扱い、現時点で bitz-env の機械フックは前2環境が対象。
- **最大のリスクは?** 少人数で複数プラットフォームと6プラグインの整合を維持できるか、実利用データが不足していること。
