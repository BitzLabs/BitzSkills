# BitzSkills 標準環境導入ガイド

BitzSkills は、BitzLabs でAIエージェントを用いて開発する際の標準作業環境です。
このガイドでは、用途に合うプロファイルを選び、Claude Code、Google Antigravity 2.0、
OpenAI Codex CLI のいずれかへ導入して、最初の開発サイクルを始めるまでを案内します。

- 目的と価値: [ミッションとビジョン](01-context/mission-vision.md)
- 対象範囲: [非目標と境界](01-context/non-goals.md)
- シナリオ別の例: [ユースケース集](ユースケース.md)

## 1. プロファイルを選ぶ

迷った場合は **標準開発**を選びます。すべてのプラグインを一括導入する必要はありません。

| プロファイル | 導入するプラグイン | 適する開発 |
|---|---|---|
| **標準開発** | `bitz-env` + `bitz-sdd` | BitzLabsの通常開発。ガードレール + 仕様駆動開発 |
| **DDD拡張** | 標準開発 + `bitz-ddd` | 複雑な業務ドメイン、集約設計、DDD成熟度評価 |
| **軽量Git** | `bitz-env` + `bitz-flow` | `.spec` を使わず、Git / GitHub規律だけを導入 |
| **スキル開発** | 標準開発 + `skill-creator` | Agent Skills の作成・検証・改善・配置 |
| **プラグイン開発** | 標準開発 + `plugin-creator` | Claude Code / Antigravity向けプラグイン開発 |

> `bitz-sdd` は現時点で `sdd-git` を含みます。同じ目的で `bitz-flow` を重複導入せず、
> `bitz-flow` は主にSDD非採用の軽量Gitプロファイルで使います。

## 2. プラグインをインストールする

### 2.1 Claude Code

標準開発プロファイル:

```
/plugin marketplace add BitzLabs/BitzSkills
/plugin install bitz-env@bitzskills
/plugin install bitz-sdd@bitzskills
```

必要に応じて拡張を追加します。

```
/plugin install bitz-ddd@bitzskills
/plugin install skill-creator@bitzskills
/plugin install plugin-creator@bitzskills
/plugin install bitz-flow@bitzskills
```

開発中のプラグインをインストールせず確認する場合:

```bash
claude --plugin-dir <BitzSkills>/plugins/<プラグイン名>
```

### 2.2 Google Antigravity 2.0

BitzSkills のローカルcloneから、標準開発プロファイルを導入します。

```bash
agy plugin validate <BitzSkills>/plugins/bitz-env
agy plugin validate <BitzSkills>/plugins/bitz-sdd
agy plugin install <BitzSkills>/plugins/bitz-env
agy plugin install <BitzSkills>/plugins/bitz-sdd
agy plugin list
```

必要な拡張は、同じ形式で `plugins/<プラグイン名>` を指定します。
`agy plugin install` が使えない環境では、各プラグインの `skills/` 配下を
`~/.gemini/config/skills/` またはプロジェクトの `.agents/skills/` へ直接配置できます。
直接配置したスキルはプラグイン管理コマンドでは更新されません。

### 2.3 OpenAI Codex CLI

標準開発プロファイル:

```bash
codex plugin marketplace add BitzLabs/BitzSkills
codex plugin add bitz-env@bitzskills
codex plugin add bitz-sdd@bitzskills
codex plugin list
```

必要な拡張は `codex plugin add <プラグイン名>@bitzskills` で追加します。
インストール後は新しいCodexセッションを開始してください。

`bitz-env` の収録スキルとAGENTS.md規律はCodexでも利用できますが、プラグイン同梱の機械フックと
`env-init` の生成対象は現在 Claude Code / Antigravity です。Codexではホストのsandbox / approvalと
AGENTS.mdを安全境界として使い、同等対応または正式な劣化モードは `SI-ENV-023` で追跡します。

## 3. プロジェクトへ標準環境を展開する

プラグインのインストールは「能力を利用可能にする」操作です。次に、対象プロジェクトへ
ガードレールとプロジェクト固有の管理ファイルを展開します。

### 3.1 ガードレールと協調環境

対象リポジトリでエージェントに次のように依頼します。

> このプロジェクトに BitzEnv の標準環境を展開してください。

Claude Code / Antigravityでは、`env-init` が危険操作を防ぐ permissions、
AGENTS.md / CLAUDE.md の規約、advisor / worker などの協調設定を対象環境に合わせて展開します。
Codexから実行する場合も既存ファイルと差分を確認できますが、Codexネイティブのフックを
生成するものではありません。
リポジトリ外へ書き込む操作は、実行前に対象を確認して承認してください。

展開後は次のように診断します。

> BitzEnv の環境を診断して、ガードレールと有効なプラグインのずれを確認してください。

`env-doctor` は読み取り中心で、設定・フック・ナラティブの同期ずれを報告します。

### 3.2 仕様駆動開発の初期化

標準開発プロファイルでは、次のように依頼します。

> このプロジェクトを BitzSDD で管理できるように初期化し、現在の要望を整理してください。

プロジェクトには次の2層を持たせます。

- `.spec/`: 要件・設計・タスク・検証状態のマスター
- `docs/`: 人間向けの意図・設計・運用ナラティブ

公開APIやスキーマを伴わない小さな変更は、spec-issue → 要件 → タスク → 実装 → 検証の
軽量レーンで進めます。新規プロダクトや大きな設計変更では、discoveryから開始します。

## 4. 日常の標準開発フロー

1. **要望を整理する** — 「この要望をspec-issueとして整理して」
2. **人間が裁定する** — accept / reject、draft要件のapproved、各ゲートは人間が決める
3. **タスクへ分解する** — `implements`、`depends_on`、`boundary` を宣言
4. **ブランチで実装する** — mainへ直接コミットしない。並列時は1エージェント=1worktree=1ブランチ
5. **検証する** — 要件からテストを導出し、実際のコマンド出力を記録
6. **報告・同期する** — `.spec` と `docs` を同期し、PRへ目的・変更点・検証結果を記載
7. **squash mergeする** — 1 PR = 1関心事を保つ

エージェントへの依頼例:

> この変更をBitzSDDの軽量レーンで仕様化し、承認が必要な地点で止めてください。

> 現在の仕様・タスク・検証状況を確認し、次に着手できる作業を教えてください。

## 5. 拡張プロファイル

### 5.1 DDD拡張

- 「ドメインストーリーを書いて」→ `ddd-story`
- 「集約と境界づけられたコンテキストを設計して」→ `ddd-model`
- 「既存コードのDDD成熟度とMMIを評価して」→ `ddd-evaluate`

成果物は `.spec/design/` に作成され、BitzSDDの設計・検証工程へ接続されます。

### 5.2 軽量Git

`.spec` を採用しないプロジェクトでは `bitz-flow` を使います。

- `flow-core`: 状況別フロー選択とConventional Commits
- `flow-worktree`: worktree並列と失敗時の安全な破棄
- `flow-pr`: Issue駆動、Draft PR、squash merge

### 5.3 スキル／プラグイン開発

`skill-creator` は作成 → 検証 → テスト → 評価 → 最適化 → 配置 → 観察改善を扱います。
`plugin-creator` は構造、コマンド、エージェント、フック、MCP、設定、スキル同梱を扱います。

## 6. アップデート

更新後はクライアントを再起動するか、新しいセッションを開始します。

### Claude Code

```bash
claude plugin update <プラグイン名>@bitzskills
claude plugin list
```

### Google Antigravity 2.0

`agy plugin install` は実体コピーです。BitzSkills のcloneを更新して再インストールします。

```bash
git -C <BitzSkills> pull --ff-only
agy plugin install <BitzSkills>/plugins/<プラグイン名>
agy plugin list
```

### OpenAI Codex CLI

Codex CLIでは、Gitマーケットプレイスのスナップショットを更新します。

```bash
codex plugin marketplace upgrade bitzskills
codex plugin list
```

プラグイン更新後、各プラグインが設定や生成物のmigration手順を提供している場合は、
そのREADMEまたは `<plugin名>:update` の案内に従ってください。

## 7. トラブルシューティング

| 症状 | 確認すること |
|---|---|
| スキルが発動しない | `claude plugin list` / `agy plugin list` / `codex plugin list` でインストール・有効状態を確認し、新しいセッションを開始 |
| ガードレールが環境間でずれる | `env-doctor` でpermissions・フック・AGENTS.md・プラグイン登録を診断 |
| `spec inspect` がFAIL | inspection reportの幽霊参照・孤児要件・frontmatter問題を確認。モノレポではworkspace一括検証を使う |
| 更新したのにAntigravityへ反映されない | install先はコピー。clone更新後に再インストールしたか確認 |
| Codexで新しいversionが見えない | `codex plugin marketplace upgrade bitzskills` 後に `codex plugin list` と新規セッションで確認 |
| SDDとGitスキルが重複する | `bitz-sdd` 利用時は内蔵 `sdd-git` を使い、軽量Git用途だけ `bitz-flow` を選ぶ |
