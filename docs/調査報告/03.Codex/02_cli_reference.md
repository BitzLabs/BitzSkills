# 2. CLI リファレンス (CLI Reference)

Codex CLI（`codex` コマンド）は、ターミナル上で自律型コーディングエージェントを操作するための主要なインターフェースです。

---

## 2.1 コマンドライン引数とサブコマンド

`codex` コマンドは、実行するモードや操作に応じてさまざまなサブコマンドやオプションを提供します。

### 基本コマンド構造
```bash
codex [SUBCOMMAND] [OPTIONS]
```

### サブコマンド詳細

| サブコマンド | 引数 | 説明 |
| :--- | :--- | :--- |
| (なし) | `[prompt]` | インタラクティブな TUI（Terminal User Interface）を起動します。初期プロンプトを指定することも可能です（例: `codex "Implement a test suite"`）。 |
| `exec` | `<prompt>` | **非インタラクティブモード（オートメーションモード）**でエージェントを動かします。スクリプトや CI/CD での実行に適しています。 |
| `resume` | `[threadId]` | 過去のセッションを再開します。`--last` オプションを指定すると、直近のセッションを自動的に再開します。 |
| `apply` | `<taskId>` | Codex Cloud で生成された特定のタスク（diff）を手元のソースコードに適用します。 |
| `sandbox` | `<platform> [cmd]`| Codex が使用する OS レベルのサンドボックス（Seatbelt や Landlock）単体でコマンドを実行してテストします。 |
| `mcp` | `<subcommand>` | Model Context Protocol（MCP）サーバーを追加、削除、表示します。 |
| `completion` | `[shell]` | 指定したシェル（`bash`, `zsh`, `fish`, `powershell`）用のコマンド補完スクリプトを生成します。 |

---

## 2.2 グローバルオプション

コマンド全体で指定可能な主要オプションです。

* **`-c, --config <key=value>`**
  - `config.toml` に設定されている値を実行時に一時的に上書きします。（例: `codex -c sandbox_mode=danger-full-access`）
* **`-i, --image <path>`**
  - エージェントに参照させる画像を渡します。画面設計のスクリーンショットや、システム構成図、エラーログ画像などを読み込ませて指示する際に使用します。
* **`--cd <directory>`**
  - エージェントの実行ルートディレクトリを指定します。現在のシェルでディレクトリを移動せずに、別プロジェクトを対象に指示を出す場合に便利です。
* **`--dangerously-bypass-approvals-and-sandbox` (別名 `--yolo`)**
  - セキュリティ保護のための「サンドボックス制限」と、ツール実行時の「人間への承認確認」をすべてバイパスして実行します。CI/CD パイプラインでの実行などで明示的にフルオートにしたい場合にのみ使用します。

## 2.3 認証とセッション管理

### 1. ログイン (`codex login`)
Codex CLI を使用するには、認証が必要です。

```bash
codex login
```
このコマンドを実行すると、ブラウザが開き、ChatGPTアカウント（Plus, Pro, Team, Enterprise）へのログインを求められます。認証に成功すると、トークンがローカル（デフォルトでは `~/.codex/auth.json`）に保存されます。

### 2. ログアウト (`codex logout`)
ローカルに保存されているログインセッション情報およびトークンを削除します。
```bash
codex logout
```

### 3. ステータス確認
現在の接続状況やログインユーザー情報を表示します。
```bash
codex status
```
または、TUIセッション内で `/status` スラッシュコマンドを実行しても同様の情報を取得できます。

## 2.4 プロジェクト初期化と実行手順

### プロジェクトの初期化
新しいリポジトリで Codex を効果的に使用するためには、AI にプロジェクトの「ルール」や「ビルド手順」を教える設定が必要です。

1. プロジェクトルートに移動し、以下を実行します：
   ```bash
   codex
   ```
2. 起動した TUI セッション内で `/init` コマンドを実行します。
3. エージェントがプロジェクトのファイル構造や使用技術（Node.js、Python、Goなど）を自動検出し、プロジェクトルールを記述する **`AGENTS.md`** または `.codex/rules` テンプレートを作成します。
4. 生成された `AGENTS.md` に、ビルドコマンドやテスト実行コマンド、コーディング規約などを人手で追加します。これにより、次回以降の実行でエージェントがそのルールを最優先で遵守するようになります。

## 2.5 TUI セッション内スラッシュコマンド

`codex` でインタラクティブセッションに入ると、チャット入力欄で `/` を入力することで、以下のメタコマンドを実行できます。

* **`/help`** — 利用可能なすべてのスラッシュコマンドの説明を表示します。
* **`/init`** — 前述のプロジェクト初期化プロセスを実行します。
* **`/new`** — 現在の会話履歴をクリアし、新しい会話スレッドを開始します（コンテキストサイズを節約するために有効）。
* **`/model`** — 使用する LLM モデルを切り替えます（例: `gpt-5.2-codex`, `gpt-5.1-codex-max`）。
* **`/compact`** — これまでの会話履歴を要約（コンパクション）し、空きトークン容量を確保します。
* **`/mode`** — エージェントの自律度（`suggest` = 提案のみ、`auto-edit` = ファイル編集の自動実行と確認、`full-auto` = 全自動）を切り替えます。
* **`/quit`** — セッションを安全に終了して CLI を抜けます。
