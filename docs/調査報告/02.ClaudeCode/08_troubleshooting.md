# 8. トラブルシューティング (Troubleshooting)

Claude Code 運用時に遭遇しやすい問題と、その切り分け・対処をまとめます。

---

## 8.1 コンテキスト溢れ・自動圧縮関連

「Context Left Until Auto-Compact: 0%」のような表示は、Microcompact で間引ける余地が尽きたサインです。放置すると強制的なフル Compact が走り、会話の細部（過去の議論の経緯など）が失われます。長時間セッションでは、区切りの良いタイミングで手動 `/compact` を挟み「何を残すか」を指示する方が、自動発火に任せるより文脈保持の質が高くなります。圧縮が3回連続で失敗するとサーキットブレーカーで自動圧縮自体が止まるため、その状態になったら `/clear` や新規セッション開始を検討します。

---

## 8.2 MCP 接続の問題

`-32000`（Generic Server Error）はほぼ常に MCP サーバープロセスのクラッシュが原因です。依存関係の欠如、Node バージョン不整合、イベントループが維持されずプロセスが即終了するケースが典型です。`claude mcp get <name>` や `claude --debug mcp` でサーバー起動コマンドを手動実行し、実際のクラッシュ理由を確認するのが最短の切り分けです。

`-32001`（タイムアウト）はセルフホストの SSE サーバーがリバースプロキシでバッファリングされているケースが多いです。OAuth 系では、HTTP MCP サーバーが「`claude mcp list` では接続済み（✓）なのに UI 上は要認証と表示される」といった状態不整合が報告されており、`/logout` → `/login` や認証キャッシュの削除・再認証が有効な回避策になります。

---

## 8.3 権限まわりのハマりどころ

「フックで `allow` を返しているのにブロックされる」「`bypassPermissions` にしたのに一部だけ拒否される」という混乱は、第6章で述べた `deny` 優先の評価順序を理解していないことに起因するケースが大半です。

フックの `matcher` は既定でツール名の完全一致または JS 正規表現として評価されるため、`code-reviewer` のようなハイフン入りのエージェント名は、バージョンによっては意図せず正規表現扱いされ `senior-code-reviewer` のような別名にもマッチしてしまいます。`^code-reviewer$` のようにアンカーを明示するのが安全です。

---

## 8.4 認証エラー

OAuth トークンは有効期限が有限で、一晩越えの自律実行や数時間の無人セッションではトークン失効により以降のすべてのツール呼び出しが 401 で失敗します。無人・長時間運用では対話的 OAuth ではなく `ANTHROPIC_API_KEY` による認証を使う方が安定します。ログイン自体が失敗する場合は、ブラウザ–CLI 間の OAuth コールバックのタイムアウト、macOS キーチェーンアクセス、DNS 解決の問題が典型的な原因で、Claude Code の更新とキャッシュクリアが第一の対処法です。

---

## 8.5 SDK 特有の落とし穴

SDK 利用時に「プロジェクト固有のルール（CLAUDE.md）を無視される」という症状は、**仕様**（`settingSources` 未指定時は設定を読み込まない）であって不具合ではありません。`settingSources: ['project']` を明示してください。また `maxTurns` を設定し忘れると、無限に近いツール呼び出しループでコストが急増するリスクがあります。SDK でツール呼び出しを監視する場合、Todo 管理が `TaskCreate`/`TaskUpdate` に分割されている（第6.5節）ことを踏まえた実装が必要です。
