# 1. 概要 (Overview)

## 1.1 コンセプトと基本設計
Claude Code は、開発者のローカルマシンまたはリモートサーバーのターミナル上で自律的に動作する、AI駆動型のコーディングアシスタントです。Anthropic の高度なフロンティアモデル（主に Claude 3.5 Sonnet）の推論能力を直接 CLI から活用し、ファイル編集、静的解析、テスト実行、Git コミットといった、コーディングにかかわる一連 of 自律的ループ（Agentic Loop）を実行します。

基本設計として、以下の思想が取り入れられています。
- **Agentic Loop**: 開発者が「~を修正して」と指示を出すと、エージェント自身が必要なファイルを探索し、内容を読み込み、修正案を書き込み、テストを実行し、エラーが発生すればその出力を分析して再修正するという自律的なループを回します。
- **Progressive Disclosure（段階的情報開示）**: トークンコストの削減とコンテキスト制限の回避のため、エージェントは最初からプロジェクト内の全ファイルを読み込むのではなく、まずディレクトリ構造やメタデータ（Skills の description 等）のみを認識し、推論の過程で必要なコンテキストを段階的に引き出します。
- **ターミナルファースト**: 特定の IDE やテキストエディタに依存せず、ターミナル上で完全に動作するよう設計されており、Vim や VS Code、リモートの SSH 接続環境でもシームレスに利用できます。

## 1.2 主な特徴
- **高度なファイル操作**: 単なるテキストの置換ではなく、文脈を理解した構造的かつ安全なファイル書き込み（Write）および編集（Edit）が行われます。
- **シェルコマンド実行**: エージェント自身がビルドやテスト、Git 操作などのシェルコマンドを必要に応じて自動実行し、その結果から次のアクションを判断します。
- **プラグインと拡張性**: 独自のコマンド、サブエージェント、ライフサイクルフック、MCP (Model Context Protocol) サーバーなどをプラグイン形式で追加し、開発ワークフローをカスタマイズできます。
- **CLAUDE.md によるプロジェクト協調**: プロジェクトルートに置いた `CLAUDE.md` ファイルを通じて、ビルド方法やテストの実行方法、コードスタイルといったプロジェクト固有の規約をエージェントに事前学習させることができます。

## 1.3 既存のコーディングアシスタントとの違い
GitHub Copilot Chat などの既存のエディタ組み込みチャットやインラインコード補完ツールとの違いは、その「自律性（Autonomy）」にあります。

| 比較項目 | 既存のコード補完・チャットツール | Claude Code |
| :--- | :--- | :--- |
| **動作形態** | 開発者がエディタでコードを書く際の補完や、質問に対する受動的回答。 | 開発者からゴールを与えられ、それを達成するために自律的に行動するエージェント。 |
| **実行ループ** | 提案されたコードを人間が手動で貼り付け、ビルドし、エラーが出たらチャットに貼り付けて聞き直す。 | ファイル読み込み、編集、ビルド、テスト実行、エラー検知と自己修正のすべてをエージェントが自動でループ実行。 |
| **環境アクセス** | 原則としてエディタで開いているファイルコンテキストのみ。 | ローカルのシェル環境、テストランナー、Git、外部MCPツールなどと直接対話。 |
| **拡張方法** | エディタの拡張機能や設定に依存。 | スキル（Skills）、スラッシュコマンド（Slash Commands）、フック（Hooks）、サブエージェントを独自プラグインで定義可能。 |

## 1.4 セキュリティとプライバシー設計
自律型エージェントである Claude Code には、強力な権限が与えられるため、セキュリティとプライバシーに関して慎重な設計がなされています。
- **Permission Mode（許可モード）**: コマンドの実行やファイルの書き込みなど、システムに影響を与える可能性のあるアクションについて、「自動許可（allow）」「拒否（deny）」「ユーザーへの問い合わせ（ask）」を細かく制御します。特に破壊的なコマンドを実行しようとする場合、キャッシュを無視して強制的にユーザーの承認（force_ask）を求めることがあります。
- **テレメトリのオプトアウト**: 利用統計データや機微なトランスクリプトの送信を拒否する設定（`DISABLE_TELEMETRY` 等）が提供されています。
- **パストラバーサル防止**: エージェントが操作するファイルパスに `..` などのパストラバーサルが含まれていないか、また `.env` などの認証情報ファイルや `/etc` などのシステムディレクトリを意図せず操作しないか、セキュリティフックなどで検証・ブロックする仕組みが組み込まれています。
