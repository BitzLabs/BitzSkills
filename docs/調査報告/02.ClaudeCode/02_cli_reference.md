# 2. CLI リファレンス (CLI Reference)

## 2.1 インストール方法
Claude Code は npm パッケージマネージャーを介してグローバルにインストールします。Node.js 環境がインストールされている必要があります。

```bash
# グローバルインストール
npm install -g @anthropic-ai/claude-code
```

> [!NOTE]
> 最新の安定版 Node.js (LTS 推奨) がローカル環境にインストールされていることを確認してください。

## 2.2 Authentication（認証）
初めて `claude` コマンドを実行する際、または認証の期限が切れた際、CLI は自動的に OAuth ブラウザフローまたはワンタイムピンを介した認証を求めます。
- **対話型認証**: ターミナルに表示される URL にブラウザでアクセスし、Anthropic アカウントでログインして認証を完了します。
- **環境変数による認証**: CI/CD 環境や非対話型のヘッドレス環境では、環境変数 `ANTHROPIC_API_KEY` または `ANTHROPIC_AUTH_TOKEN` を設定することで、認証フローをスキップできます。

```bash
# API キーを環境変数に設定して起動する場合
export ANTHROPIC_API_KEY="your-api-key-here"
claude
```

認証情報や設定データは、グローバルユーザーディレクトリ `~/.claude/` 配下に暗号化、または制限されたパーミッションで保持されます。

## 2.3 基本的な実行方法
Claude Code は主に 2 つの実行モードを持ちます。

### 1. 対話型セッションモード (Interactive Mode)
単に `claude` と打って実行します。このモードでは専用のプロンプトシェルが起動し、Claude と対話しながら連続的なタスクを依頼できます。

```bash
$ claude
Welcome to Claude Code. Type /help for available commands.
project-root >
```

### 2. ワンショットモード (One-shot Mode)
`-p` オプションを使用して、特定のタスクを一度だけ実行して終了させます。スクリプトや CI パイプラインに組み込む際に便利です。

```bash
# 特定のタスクを単発で実行し、終了コードを伴って終了する
claude -p "src/index.ts のコード品質をレビューして、もし修正が必要なら提案してください"
```

## 2.4 CLI 引数とオプション
`claude` コマンドの起動時に、挙動を制御するためのオプションが指定可能です。

- **`-p "[prompt]"`**: 指定したプロンプトをワンショットで実行します。
- **`--model [model_name]`**: 使用する AI モデルを指定します（例: `sonnet` (標準), `haiku` (高速軽快), `opus` (複雑な推論)）。環境変数 `ANTHROPIC_MODEL` よりも優先されます。
- **`--plugin-dir [path]`**: 開発中のプラグインのパスを指定して起動し、そのセッションでプラグインをロードします。
- **`--mcp-config [path]`**: 使用する Model Context Protocol (MCP) の設定ファイルのパスを指定します。デフォルトは `./.mcp.json` です。
- **`--disallowed-tools [tools]`**: 特定の組み込みツールの使用をエージェントに禁止します。コンマ区切りで複数指定（例: `--disallowed-tools Write,Bash`。すべて禁止する場合は `*`）。
- **`--verbose` / `--debug`**: 実行時のデバッグログ、フックの読み込み状況、モデルとツール間の詳細な JSON ペイロードをコンソールに出力します。

## 2.5 主要なサブコマンド
対話型セッション内でスラッシュ（`/`）で始まる特別なコマンドを実行し、エージェントの操作や設定を変更できます。

- **/compact**: これまでの会話履歴を要約し、メモリコンテキストを圧縮してトークンコストを節約します。長時間の開発セッションで有効です。
- **/clear** (または **/reset**, **/new**): 現在の会話コンテキストと履歴を完全に消去し、クリーンな状態から新しいセッションを開始します。
- **/model [model_name]**: 会話の途中で使用するモデルを動的に切り替えます（例: `/model haiku`）。
- **/diff**: 現在のセッション中にエージェントが行ったファイルの変更箇所を、インタラクティブな差分ビューアで確認します。
- **/usage** (or **/cost**, **/stats**): 現在のセッションのトークン消費量、累積コスト、API 制限に対する現在の使用状況の統計を表示します。
- **/permissions**: 許可（allow / deny / ask）ルールの一覧と現在のパーミッションモードを確認・編集します。
- **/agents**: 定義済みのサブエージェントを一覧・管理します。
- **/hooks**: 構成済みのフックを読み取り専用ブラウザで確認します。イベントごとの登録数・マッチャー・各ハンドラの詳細（コマンド／プロンプト／URL）と、どの設定ファイル由来かを表示します（詳細は第4章 4.5 を参照）。
- **/doctor**: インストール状態・認証・設定・依存関係を診断し、問題があれば指摘します。
- **/config**: 設定管理のためのインタラクティブ UI を起動します。また `/config key=value`（例：`/config verbose=true`）の形式でクイックに変更することも可能です。
- **/plugin**: プラグインの追加・削除・インストールを行います。
  - `/plugin marketplace add [path/repo_url]` : 指定したリポジトリまたはディレクトリをマーケットプレイスとして追加します。
  - `/plugin install [plugin_name]@[marketplace]` : マーケットプレイスから特定のプラグインをインストールします。
