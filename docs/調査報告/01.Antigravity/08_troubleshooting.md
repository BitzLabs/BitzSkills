# 8. トラブルシューティング (Troubleshooting)

Google Antigravity の運用、特に Claude Code 互換プラグインを持ち込む際に遭遇しやすい問題をまとめます。

---

## 8.1 互換レイヤーで検出されない配置（最大の落とし穴）

| Claude Code 側の配置 | Antigravity での結果 | 正しい配置 |
| --- | --- | --- |
| `hooks/hooks.json` | **検出されない**（サイレントに無視） | プラグインルート直下の `hooks.json` |
| `.mcp.json` またはマニフェストの `mcpServers` | **検出されない** | プラグインルート直下の `mcp_config.json` |
| `.claude-plugin/plugin.json` | **読み込まれない** | プラグインルート直下の `plugin.json` |

いずれも「エラーにはならず、単に機能が有効化されない」ため、動作確認をせずに配布すると気づきにくい問題です。第7.5節の二重検証を徹底してください。

---

## 8.2 `install` = 実体コピーの罠

`agy plugin install <dir>` は対象ディレクトリの**そのままコピー**＋ `~/.gemini/config/import_manifest.json` への登録であり、シンボリックリンクではありません。開発中にプラグインを更新しても配置先には反映されず、次のような症状が起きます。

- スキル／フックを修正したのに動作が変わらない → 実は古いコピーを見ている

対策として、開発中は `--plugin-dir <path>` で直接読み込む（インストール不要の検証モード）を使い、確認が終わってから `install` します。

---

## 8.3 名前空間衝突

同名のスキル／プラグインが複数の発見レイヤーに存在すると、5段階優先順位（第6.2節）に従い上位が勝ち、下位は**警告なくサイレントに上書き**されます。デバッグ時は `agy plugin list` に加え、各 customization root を手動で確認する必要があります。

MCP ツール名の名前空間 `mcp__plugin_<plugin_name>_<server_name>__<tool_name>` は長い文字列になるため、複数プラグイン導入時にツール一覧が肥大化しやすくなります。事前許可リストはワイルドカードでなく具体名で書くことが推奨されます。

---

## 8.4 プロキシ環境

一部バージョンで `HTTP_PROXY` / `HTTPS_PROXY` / `ALL_PROXY` を正しく認識しない挙動が報告されています（エンタープライズ社内ネットワーク経由でタイムアウト・SSL 疎通エラーが発生しうる）。

> [!NOTE]
> この既知問題は、本リポジトリのワークスペース資料には記載がある一方、今回の外部検索では Antigravity 固有の一次情報として直接の裏取りができませんでした。断定は避け「報告されている可能性がある」程度に扱うのが安全です。Cloud Shell 等のマネージド環境ではプロキシは透過的に処理されます。

---

## 8.5 起動元によるフックスクリプトの差異

同じフックスクリプトを CLI・IDE・Antigravity 本体で使い回すと、共通入力の `transcriptPath` / `artifactDirectoryPath` のディレクトリ名が起動元ごとに変わる（第6.3節）ため、これらのパスを固定文字列で前提にしているスクリプトが破綻します。パスは入力 JSON から動的に取得してください。
