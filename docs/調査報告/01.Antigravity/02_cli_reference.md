# 2. CLI リファレンス (CLI Reference)

## 2.1 `agy` コマンド概要
Antigravity CLI は、ターミナル上で自律型エージェントと直接やり取りするための Go 製のツールです。コマンド `agy` をプロジェクトのワークスペース（または任意のディレクトリ）で実行すると、インタラクティブな Terminal User Interface (TUI) が起動します。

## 2.2 起動とオプション

### 基本起動
```bash
agy
```
カレントディレクトリを起点とし、上位へ遡ってリポジトリルート（`.git` や `.agents/` ディレクトリが存在する場所）を探索し、その範囲のプロジェクト文脈を取り込んだ状態で TUI を開始します。

### 主な起動オプション（コマンドライン引数）
*   `--sandbox=<true|false>`: ターミナルの隔離実行サンドボックスの有効/無効を切り替えます（デフォルトは `true`）。
*   `--model=<model_name>`: 起動セッションで使用するモデルを明示的に指定します。2026年時点では Gemini 3 系を主軸に、Claude Sonnet/Opus・GPT-OSS 等を含むマルチモデルから選択できます。デフォルトは `"auto"` または設定ファイルで定義されたモデルです。
*   `--verbosity=<low|high>`: エージェントのツール呼び出しプロセスやステップ表示のログ出力量を調整します。
*   `--plugin-dir <path>`: 開発中のプラグインをローカル検証用として、インストールなしに直接読み込んで TUI を起動します（Claude Code の `--plugin-dir` に相当する検証オプション）。

## 2.3 プラグイン管理サブコマンド
TUI の起動とは別に、`agy` CLI 自体にプラグインを管理するための以下の各種サブコマンドが用意されています。

```bash
# インポート・インストール済みプラグイン一覧の表示
agy plugin list

# 指定したローカルのプラグインディレクトリをグローバル（~/.gemini/config/plugins/）へコピーして登録
agy plugin install <ローカルプラグインディレクトリのパス>

# プラグインのアンインストール
agy plugin uninstall <プラグイン名>

# 指定したプラグインの有効化 / 無効化
agy plugin enable <プラグイン名>
agy plugin disable <プラグイン名>

# プラグイン構造の検証（マニフェスト、フック、スキル、エージェント定義が規格に適合しているかチェック）
# Claude Code 形式のレイアウトを検知して検証することも可能です。
agy plugin validate [path]

# 他環境（Gemini CLI または Claude Code）からのプラグイン定義のインポート
agy plugin import [gemini|claude]
```

> [!NOTE]
> `agy plugin install` の実態は、対象ディレクトリの**そのままのコピー**と、グローバル管理台帳ファイルである `~/.gemini/config/import_manifest.json` への登録です。元ディレクトリと配置先はシンボリックリンクではなく実体コピーされるため、プラグイン開発時に更新を反映させるには再インストール（または `--plugin-dir` での起動）が必要です。

## 2.4 TUI インタラクティブ・スラッシュコマンド
`agy` 起動後に起動する TUI セッション内で、プロンプトに `/` から始まる特定のコマンドを直接入力することで、各種管理機能にアクセスできます。

*   **/config または /settings**:
    設定メニューを TUI 上に対話的に開きます。現在の `settings.json` の内容を画面上で確認し、編集・適用できます。
*   **/mcp**:
    現在有効な MCP サーバーの一覧、各サーバーが提供するカスタムツールの一覧、その入力スキーマ（引数仕様）および詳細な説明を確認できます。
*   **/keybindings**:
    TUI 上でのショートカットキーなどのカスタムキーバインドの設定や確認を行います。
*   **/logout**:
    現在保存されている認証資格情報をキーリングから削除し、ログアウトします。
*   **/clear**:
    チャット画面のバッファおよび履歴コンテキストをクリアし、セッションを初期化します。
*   **/help**:
    利用可能なスラッシュコマンドやショートカット、基本的な操作方法を一覧表示します。

## 2.5 認証フロー (Authentication Flow)
Antigravity CLI は安全な認証およびトークンストアを提供します。

### 1. ローカルブラウザによる OAuth 認証
初回起動時、または認証情報が切れている場合、CLI は OS の標準ブラウザを起動し、Google アカウントによるログインと Antigravity プラットフォームへのアクセス権限認可（OAuth フロー）を求めます。
認証が完了すると、取得した認証トークンは OS の**システムキーリング（Windows Credential Manager, macOS Keychain, Linux secret-service など）**に安全に保存されます。

### 2. リモート・SSH 環境での認証（ヘッドレスモード）
ブラウザがインストールされていないリモートサーバーや SSH 接続先のコンテナ環境などで `agy` を実行した場合、CLI はブラウザの自動起動をスキップし、ターミナル上に認証用の一時 URL と検証用コード（デバイスコード）を表示します。
開発者はローカル PC のブラウザでその URL にアクセスして検証コードを入力し、認証を完了させることができます。
