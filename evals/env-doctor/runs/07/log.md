## 判定

### (a) 「permissions をいじったから整合を確認したい」

**判定: 使う**

根拠: description の後半「またはガードレール・**permissions を変更した後**に使用する」に
文字通り合致する。ユーザー発話は「permissions をいじった（＝変更した）」＋
「整合を確認したい」で、トリガー条件の言い換えそのもの

### (b) 「この Python コードの健全性をチェックして」

**判定: 使わない**

根拠: description のトリガー語「環境の健全性チェック」は**環境**（.claude/settings.json・
AGENTS.md・レジストリ等）の健全性を指しており、対象は「開発環境」であって
アプリケーションコードではない。「Python コードの健全性」は一般的なコード品質・
静的解析・バグチェックの文脈であり、description が扱う3層（permissions/フック/AGENTS.md）
・レジストリ・委譲マトリクスのいずれにも該当しない。「健全性チェック」という字面の
部分一致だけで誤発動させないという判定
