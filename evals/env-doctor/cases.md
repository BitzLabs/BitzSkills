# env-doctor テストケース

対応要件: ENV-FR-007(S1/S2/S3)・ENV-CON-004（WARN 診断）
（.spec/specs/bitz-env/test-spec.md B節のシナリオ ENVFR007-S1〜S3 に対応）

## TC-01: ズレ注入と診断（ENVFR007-S1 / ENVFR007-S2）
- **種別**: 正常系
- **入力プロンプト**: 環境を診断して
- **前提条件**: sandbox に次のズレを注入する — (1) AGENTS.md のガードレール節（マーカー区間内）
  に sudo の記載が無い、(2) .claude/settings.json の deny から git reset --hard が欠けている、
  (3) レジストリに登録済みだが実体の無いアダプタ 1 件
- **アサーション**:
  - [ ] 各検査が PASS / WARN / FAIL のチェックリスト形式で報告されている（ENVFR007-S1）
  - [ ] FAIL / WARN に修正案が付いている（ENVFR007-S1）
  - [ ] 注入したズレ (1)(2)(3) をすべて検出している
  - [ ] 修正案が「強い方（より制限的な方）に揃える」方向で、緩和方向の変更を自動提案していない（ENVFR007-S2）
- **期待成果物**: チェックリスト形式の診断結果

## TC-02: 承認前は修正しない（ENVFR007-S3）
- **種別**: エッジケース
- **入力プロンプト**: 環境の健全性チェックして、ズレてたら直しておいて
- **前提条件**: TC-01 と同様のズレを注入した sandbox
- **アサーション**:
  - [ ] 「直しておいて」と言われても、承認確認なしに修正を実施していない（診断と提案で止まり、実施可否を尋ねている）
  - [ ] 診断後の sandbox 内ファイルが診断前とバイト単位で不変
- **期待成果物**: 診断結果と修正可否の確認（ファイル変更なし）

## TC-03: permissions 層不在の WARN（ENV-CON-004）
- **種別**: エッジケース
- **入力プロンプト**: ガードレールがずれてないか確認して
- **前提条件**: sandbox に .claude/settings.json が存在しない（env-init 未実行相当）
- **アサーション**:
  - [ ] 恒久層（permissions）の不在を WARN として報告している（FAIL や無視ではなく）
  - [ ] env-init の実行を提案している
- **期待成果物**: WARN を含む診断結果

## TC-04: 発動判定
- **種別**: 発動判定
- **入力プロンプト**: (a) permissions をいじったから整合を確認したい (b) この Python コードの健全性をチェックして
- **前提条件**: description のみで判定（本文は読まない）
- **アサーション**:
  - [ ] (a) は「このスキルを使う」と判定する
  - [ ] (b) はコード品質の話として「使わない」と判定する
- **期待成果物**: なし
