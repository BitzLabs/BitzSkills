# run 07 — TC-02（非準拠アダプタの拒否）

- ケースID: TC-02
- モード: スキルあり（env-register SKILL.md を読了して遂行）
- 実行日時: 2026-07-12

## 手順の記録

1. 入力プロンプト「この my-tool プラグインも委譲先として登録して」を受領。
2. SKILL.md「1. アダプタの検出」に従い、プロンプトにプラグイン名が含まれているため
   `plugins/my-tool/plugin.json` を直接確認した。
3. 内容は `{"name": "my-tool", "version": "1.0.0", "description": "汎用ツールプラグイン（協調アダプタではない）"}`
   のみで、`metadata.collab` が存在しない。SKILL.md の契約チェック基準
   （「collab.agent があること」「delegate 相当の役割スキルを持つこと」）のいずれも満たさない。
4. 「非準拠なら理由を報告し、登録しない」（SKILL.md 1節末尾）に従い、レジストリへの登録・
   CLAUDE.md の更新のいずれも行わずに終了した。
5. ユーザーへの報告（シミュレート）: 「my-tool は `metadata.collab`（能力宣言）を持たず、
   delegate 相当の役割スキルも宣言していないため、契約（v2）非準拠と判定しました。
   協調アダプタとしては登録できません。もし委譲先として使いたい場合は、
   my-tool 側で collab 契約に対応してもらう必要があります」。
6. ファイル操作は一切行っていない（sandbox は original と完全一致）。

## アサーション結果

- [x] ✅ 非準拠の理由（能力宣言なし・delegate 役割なし）を具体的に報告している
      証拠: 上記5の報告内容（`metadata.collab` 不在・delegate 役割スキル不在を明示）
- [x] ✅ レジストリに登録していない
      証拠: `diff -rq runs/07/sandbox runs/07/original` が差分なし（`.claude/` は生成されていない）

## 気づき

- 契約チェックの基準（collab.agent の有無・delegate 役割の有無）が SKILL.md 内に明文化されており、
  判定に迷う余地がなかった。ベースライン（run02）ではこの基準を知らず誤って登録してしまっており、
  スキルの効果が明確に確認できた。
- SKILL.md には「非準拠の場合の報告フォーマット」までは規定されていない（自由記述）。
  ケースによって報告の粒度がぶれる可能性があり、テンプレート化を検討する余地がある。
