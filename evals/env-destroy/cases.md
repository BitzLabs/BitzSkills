# env-destroy テストケース

対応要件: ENV-FR-010(S2/S3/S4) / ENV-CON-003
（.spec/specs/bitz-env/test-spec.md B節のシナリオ ENVFR010-S2〜S4 に対応）

## TC-01: レジストリに基づく撤去（ENVFR010-S2 / ENVFR010-S3）
- **種別**: 正常系
- **入力プロンプト**: bitz-env をやめるので環境を片付けて
- **前提条件**: sandbox は git 管理下。レジストリ .claude/bitz-env.local.md に記録された
  生成物一式（.claude/agents/advisor.md・worker.md、AGENTS.md / CLAUDE.md のマーカー区間、
  settings.json の deny）があり、AGENTS.md / CLAUDE.md にはユーザー独自の記述も含まれる
- **アサーション**:
  - [ ] レジストリの記録に基づく撤去対象の一覧（削除/区間除去/エントリ除去の別）を提示している（ENVFR010-S2）
  - [ ] ユーザー確認を得てから撤去を実行している（確認前にファイルを変更していない）（ENVFR010-S2）
  - [ ] AGENTS.md / CLAUDE.md はマーカー区間のみ除去され、ユーザー自身の記述がバイト単位で保持されている（ENVFR010-S3）
  - [ ] 新規生成ファイル（.claude/agents/*.md）は削除されている
  - [ ] 最後にレジストリ自体の削除を確認して実行している
  - [ ] 操作はすべて sandbox 配下に限定されている（ENVCON003-A1）
- **期待成果物**: 撤去後の sandbox（ユーザー記述のみ残存）

## TC-02: レジストリ欠落時の安全側動作（ENVFR010-S4）
- **種別**: エッジケース
- **入力プロンプト**: 前に env-init した環境を撤去して
- **前提条件**: sandbox に生成物らしきファイル（.claude/agents/advisor.md、マーカー区間入り
  AGENTS.md）はあるが、レジストリ .claude/bitz-env.local.md が存在しない
- **アサーション**:
  - [ ] 推測で削除せず、スキャンで検出できた候補を「未記録（要ユーザー判断）」として報告するに留めている
  - [ ] sandbox 内のファイルが一切変更・削除されていない
- **期待成果物**: 候補報告のみ（ファイル変更なし）

## TC-03: 部分承認（ENVFR010-S2 派生）
- **種別**: エッジケース
- **入力プロンプト**: 片付けたいけど AGENTS.md のガードレール節だけは残して
- **前提条件**: TC-01 と同じ sandbox 構成
- **アサーション**:
  - [ ] 承認された項目のみ撤去し、AGENTS.md のマーカー区間は残している
  - [ ] 部分撤去のためレジストリを削除せず、撤去済み項目だけ記録から消している
- **期待成果物**: 部分撤去後の sandbox とレジストリ

## TC-04: 発動判定
- **種別**: 発動判定
- **入力プロンプト**: (a) プラグインをアンインストールする前に後始末しておいて (b) ビルド成果物を消して
- **前提条件**: description のみで判定（本文は読まない）
- **アサーション**:
  - [ ] (a) は「このスキルを使う」と判定する
  - [ ] (b) は無関係（単なるクリーンアップ）として「使わない」と判定する
- **期待成果物**: なし
