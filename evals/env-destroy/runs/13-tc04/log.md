## 参照した description（全文）

> env-init が対象プロジェクトへ展開した生成物（settings.json の permissions・
> AGENTS.md / CLAUDE.md のマーカー区間・.claude/agents/ の advisor / worker）を、
> レジストリの記録に基づきユーザー確認付きで安全に撤去する。「bitz-env を片付けて」
> 「環境を撤去して」「アンインストールの後始末」「生成物を消して」「env-destroy」と
> 言われたとき、または bitz-env プラグインを無効化・アンインストールする前後に
> 使用する。環境の展開は env-init、診断は env-doctor が担当する。

## 判定

### (a) 「プラグインをアンインストールする前に後始末しておいて」

**判定: このスキルを使う**

理由:
- description の例示フレーズ「アンインストールの後始末」とほぼ同義・同文脈
- 「bitz-env プラグインを無効化・アンインストールする前後に使用する」というトリガー条件に
  文字通り合致（「アンインストールする前に」＝アンインストール前後の一致）
- 「プラグイン」という語が bitz-env に限定されているわけではないが、
  env-destroy が扱う対象（settings.json permissions・AGENTS.md/CLAUDE.md マーカー区間・
  .claude/agents/ の advisor/worker）はまさに「プラグイン導入で展開された生成物」の
  典型例であり、他にこの語彙域をカバーするスキルは無いと判断した

### (b) 「ビルド成果物を消して」

**判定: このスキルを使わない**

理由:
- 「ビルド成果物」は env-destroy が扱う対象（bitz-env の生成物・permissions・
  マーカー区間・advisor/worker）と無関係。単なる `dist/` `build/` 等の
  クリーンアップであり、description のどのトリガーフレーズにも合致しない
- description 末尾に「環境の展開は env-init、診断は env-doctor が担当する」と
  隣接スキルとの分担が明記されているが、ビルド成果物削除はそのいずれの
  管轄でもなく、bitz-env プラグイン体系の外側の一般的な作業と判断した
