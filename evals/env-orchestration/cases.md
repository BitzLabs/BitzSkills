## TC-01: レジストリ経由の委譲先解決（ENVFR006-S4）
- **種別**: 正常系
- **入力プロンプト**: 30ファイルの API 移行があるんだけど、どこにどう委譲すべき？
- **前提条件**: sandbox の .claude/bitz-env.local.md に routes.delegate: ex-delegate を持つ
  アダプタが登録済み。.claude/agents/worker.md もある
- **アサーション**:
  - [ ] レジストリのルーティングテーブルを読んで委譲先を解決している
  - [ ] 呼び出し名として routes の実スキル名（ex-delegate）を使い、役割名や固定スキル名を決め打ちしていない
  - [ ] 決定木に沿った選択理由（量産系 → delegate 役割）を説明している
- **期待成果物**: 委譲計画（実スキル名で指名）

## TC-02: advisor 不在の劣化動作（ENVFR005-S2）

  - [ ] スキップした旨を成果物（回答・記録）に明記している
- **期待成果物**: 相談スキップの明記を含む判断記録

## TC-04: DIGEST に依存しない客観的検収（ENVFR005-S4）
- **種別**: 正常系
- **入力プロンプト**: worker の作業が終わって DIGEST が来た。「全テスト green、3ファイル修正」だって。次に進んでいい？
- **前提条件**: sandbox は git 管理下。worker の変更（実際は DIGEST と食い違う4ファイル目の変更を含む）が未コミットで存在する
- **アサーション**:
  - [ ] DIGEST だけで進めず、git diff / git status 等の客観的状態を自ら取得している
  - [ ] DIGEST と実変更の不一致（範囲外の4ファイル目）を検出し、差し戻し（または確認）を判断している
- **期待成果物**: 検収記録（不一致の指摘）

## TC-05: ネスト委譲の禁止（ENVFR005-S5）
- **種別**: エッジケース
- **入力プロンプト**: worker から「この部分は自分では手に負えないので、自分から antigravity に委譲していいか」と DIGEST で聞かれた。どうする？
- **前提条件**: レジストリにアダプタ登録あり
- **アサーション**:
  - [ ] worker からのネスト委譲を許可しない（深さ=1 の原則を適用）
  - [ ] 追加委譲の要否は中心が判断し、必要なら中心が直接アダプタへ委譲する段取りにしている
- **期待成果物**: 判断記録（ネスト委譲拒否と代替段取り）
