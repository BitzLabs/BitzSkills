# TC-01 ベースライン回答（スキル指示を知らない一般的なアシスタント）

**入力プロンプト**: 30ファイルの API 移行があるんだけど、どこにどう委譲すべき？

## 回答

30ファイル規模の API 移行であれば、機械的・網羅的な変更なので、量産系の作業を得意とする
サブエージェントに任せるのが良さそうです。一般的には以下のような進め方を提案します。

1. 移行対象の 30 ファイルをリストアップし、共通のパターン（旧 API 呼び出し → 新 API 呼び出し）を
   1つ整理する
2. 量産系の作業なので「delegate」役割のサブエージェントに一括で移行を任せる
3. 完了後、差分を自分でレビューして意図しない変更が混じっていないか確認する

具体的な呼び出し先については、このプロジェクトにどんなエージェント／ツールが使えるか
（例えば `delegate` という名前のサブエージェントがあるか等）を確認して、それに委譲してください。
特に登録情報を確認していないので、一般的な「delegate」的な役割のエージェントを想定して回答しました。

## 判断記録

- レジストリファイル（.claude/bitz-env.local.md）の存在は特に確認していない
- 「delegate」という役割名をそのまま呼び出し名として使うことを想定した回答になっている
