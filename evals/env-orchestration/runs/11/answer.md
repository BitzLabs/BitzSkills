# TC-06 発動判定（description のみを根拠に判定、本文は読まない）

## 根拠にした description（全文）
> bitz-env のモデル非依存な協調運用（委譲型・相談型・合議型）を実行時に案内するスキル。
> 「どのモデルに委譲すべきか」「advisor に相談したい」「合議で決めたい」
> 「クロスモデルで検証したい」「協調パターンを教えて」と言われたとき、または
> 大きな設計判断・破壊的変更・原因不明の不具合で単独判断を避けたいときに使用する。
> 環境の展開は env-init、アダプタの登録は env-register が担当する。

## (a) この不具合調査、どのモデルに任せるべきか迷ってる
- 判定: **このスキルを使う**
- 根拠: description のトリガー文言「どのモデルに委譲すべきか」に直接一致し、
  かつ「原因不明の不具合で単独判断を避けたいとき」にも該当する。env-orchestration の
  管轄と明確に判定できる。

## (b) git のブランチ運用ルールを教えて
- 判定: **使わない**
- 根拠: description のどのトリガー文言（委譲/相談/合議/クロスモデル検証/協調パターン/
  設計判断/破壊的変更/原因不明の不具合）にも該当しない。ブランチ運用ルールは
  Git/GitHub フローの話題であり、env-orchestration の管轄外（sdd-git 等の管轄と判断）。

## 判定結果まとめ
| 入力 | 判定 |
| --- | --- |
| (a) どのモデルに任せるべきか | 使う |
| (b) ブランチ運用ルール | 使わない |
