# TC-03 スキルあり回答（env-orchestration の指示に従う）

**入力プロンプト**: この破壊的変更は影響が大きいから合議で決めて

## 前提の読み込み
- `.claude/bitz-env.local.md` を読んだ: routes.advisor: `advisor`、review 役割のアダプタ登録なし
  （routes.review: []）
- `.claude/agents/advisor.md`, `.claude/agents/worker.md` あり

## パターン選択の決定木の適用
- 破壊的変更・影響大 → 【合議型】advisor + アダプタの review 役割（ルーティングテーブルで解決）へ
  並列諮問、が本来の選択

## 合議メンバーの確定
- advisor: 登録あり → 成立
- アダプタの review 役割: レジストリに登録なし（routes.review が空）→ 不成立
- worker: 合議メンバーとして直接カウントしない（決定木上は「必要なら worker」の位置づけで
  今回の議題は設計判断のため対象外）
- 結果: 合議として成立するメンバーは **advisor 1名のみ**

## 劣化動作の適用
「合議の格下げ（1名なら相談型、0名なら単独判断）」を適用する。
メンバーが1名しか成立しないため、**合議型から相談型へ格下げ**して advisor 1名への
相談として実施する。

## 判断記録
- 合議メンバーとして成立するのは advisor 1名のみであることを確認した（review 役割の
  アダプタ登録なし）
- **合議型 → 相談型へ格下げした旨を成果物に明記**: 「本来は合議型だが、成立メンバーが
  advisor 1名のみのため相談型へ格下げして実施」
- 格下げ後、advisor へ議題・判断基準・回答形式をまとめて相談する段取りとする
