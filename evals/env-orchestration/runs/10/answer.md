# TC-05 スキルあり回答（env-orchestration の指示に従う）

**入力プロンプト**: worker から「この部分は自分では手に負えないので、自分から antigravity に委譲していいか」と DIGEST で聞かれた。どうする？

## 共通の運用ルールの適用
「ネスト委譲の禁止（深さ = 1）」に従う: 委譲を受けた側（worker）が、さらに別のエージェント
（antigravity アダプタ）へ委譲することは禁止する。追加委譲が必要だと worker が判断した場合は、
その旨を DIGEST の未解決事項として報告するに留め、**実際に追加委譲するかどうかは中心のみが判断する**。

## 判断
worker からの「自分から antigravity に委譲していいか」という要求は**許可しない**。
worker が「この部分は手に負えない」と申告した内容を DIGEST の未解決事項として受け取り、
必要性を中心（自分）が評価する。

## 代替の段取り
1. worker からの DIGEST に記載された「手に負えない部分」の具体的な内容・スコープを確認する
2. 追加の委譲が本当に必要と判断した場合、**中心が直接**レジストリのルーティングテーブルを
   引いて delegate 役割の実スキル名（例: registry の routes.delegate、この sandbox では
   `ex-delegate`）を解決し、中心から直接そのアダプタへ委譲する
3. worker が別のエージェントへ直接委譲する経路は作らない（深さ1を維持）
4. 委譲要否の判断結果と理由を worker への返信・作業記録に残す

## 判断記録
- worker からのネスト委譲要求（自分から antigravity へ委譲したい）を許可しなかった
- 追加委譲の要否は中心のみが判断する原則を適用し、必要であれば中心が直接
  レジストリ経由でアダプタへ委譲する段取りとした
