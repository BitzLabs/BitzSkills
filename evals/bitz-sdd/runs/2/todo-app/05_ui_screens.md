# 画面設計

## 画面構成（1ページ構成 / SPA風）

### 画面: タスク一覧・追加画面（トップ画面を兼ねる）

```
┌──────────────────────────────┐
│  ToDo リスト                   │
├──────────────────────────────┤
│ [新規タスク入力欄] [追加ボタン]   │
├──────────────────────────────┤
│ □ 牛乳を買う           [削除]  │
│ ☑ 資料をまとめる (完了)  [削除] │
│ □ 部屋を掃除する        [削除]  │
└──────────────────────────────┘
```

## 画面要素と挙動
- チェックボックス: クリックで `PATCH /api/tasks/:id` を呼び、完了/未完了を切り替える。
  完了済みタスクは取り消し線＋グレー表示にする。
- 追加ボタン: 入力欄が空でない場合のみ有効化。`POST /api/tasks` を呼び、
  成功したら入力欄をクリアして一覧を再取得する。
- 削除ボタン: 確認ダイアログを出さず即時 `DELETE /api/tasks/:id`（個人用途のため簡略化）。
- 初期表示: ページロード時に `GET /api/tasks` で全件取得し描画する。

## 画面遷移
今回は一覧・追加・完了操作すべてを1画面に集約する（個人用の小規模アプリのため、
画面遷移を増やさずモーダルやページ遷移なしで完結させる）。
