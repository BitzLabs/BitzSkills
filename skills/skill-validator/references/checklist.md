# スキル検査チェックリスト

`skill-validator` が使用する検査項目の正式な一覧。各項目に判定基準と判定方法を示す。
仕様の出典は [Agent Skills Specification](https://agentskills.io/specification)。

## A. 構造

| # | 項目 | 判定基準 |
| --- | --- | --- |
| A1 | SKILL.md の存在 | スキルフォルダ直下に `SKILL.md` がある |
| A2 | frontmatter の形式 | ファイル先頭が `---` で始まり、対応する `---` で閉じ、間がYAMLとしてパースできる |
| A3 | 推奨ディレクトリ名 | 追加ファイルが `scripts/` `references/` `assets/` のいずれか（または明確な意図のあるディレクトリ）に置かれている |

## B. name

| # | 項目 | 判定基準 |
| --- | --- | --- |
| B1 | 存在 | `name` フィールドがある（省略可の実装もあるが、互換性のため必須扱いとする） |
| B2 | 文字数 | 1〜64文字 |
| B3 | 文字種 | 英小文字・数字・ハイフンのみ（正規表現: `^[a-z0-9]+(-[a-z0-9]+)*$`。これで先頭末尾ハイフン・連続ハイフンも同時に検査できる） |
| B4 | フォルダ名との一致 | `name` の値が親フォルダ名と完全一致する |

## C. description

| # | 項目 | 判定基準 |
| --- | --- | --- |
| C1 | 存在 | `description` フィールドがあり空でない |
| C2 | 文字数 | 1〜1024文字 |
| C3 | 「何をするか」 | スキルが行う作業・成果が読み取れる |
| C4 | 「いつ使うか」 | 発動すべき状況・ユーザーの発言例・キーワードが読み取れる |

C3/C4は機械判定できないため、読み取れない場合は根拠を添えて判定する。

## D. 本文

| # | 項目 | 判定基準 |
| --- | --- | --- |
| D1 | 行数 | SKILL.md 全体が500行未満（推奨。超過は❌ではなく⚠️とし、references/への分離を提案） |
| D2 | 相対パス参照の実在 | 本文中で参照している `references/...` `scripts/...` `assets/...` が実在するファイル/ディレクトリを指す |
| D3 | 参照の深さ | ファイル参照がスキルフォルダからできるだけ1階層以内にとどまっている（超過は⚠️） |

## E. 設計品質（⚠️判定のみ・断定しない）

| # | 項目 | 判定基準 |
| --- | --- | --- |
| E1 | 単一責務 | 2つ以上の独立した作業（例: 「作成」と「デプロイ」）を1スキルで扱っていない |
| E2 | 冗長性 | 同じ内容の繰り返し、本文に埋め込まれた長大な例など、references/へ分離すべき内容がない |
| E3 | 安全確認 | コード実行・ファイル削除・ネットワークアクセスを伴う指示に、安全確認・検証の手順が明記されている |

## F. metadata（ライフサイクル管理・このライブラリの規約）

| # | 項目 | 判定基準 |
| --- | --- | --- |
| F1 | version | `metadata.version` が存在し、`X.Y.Z` 形式（正規表現: `^\d+\.\d+\.\d+$`）で引用符付き文字列である |
| F2 | author | `metadata.author` が存在し空でない |
| F3 | 日付 | `metadata.created` と `metadata.updated` が存在し `YYYY-MM-DD` 形式。updated が created より過去でない |
| F4 | installed-* の混入禁止 | ライブラリ内のスキルに `metadata.installed-at` / `metadata.installed-from` が**無い**こと（あれば配置先からの手動逆コピーの痕跡） |

F群の規約の正式な定義は skill-creator の spec.md（metadata運用規約）、
installed-* の扱いは skill-packager の lifecycle.md にある。

## 判定記号の使い分け

- ✅ 基準を満たす
- ❌ 機械的に判定できる基準（A・B・C1・C2・D2）を満たさない
- ⚠️ 判断の余地がある項目（C3・C4・D1・D3・E群）で懸念がある。根拠を添えて
  ユーザーの判断を仰ぐ
