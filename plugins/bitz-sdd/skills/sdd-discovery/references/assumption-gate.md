# 仮説検証ゲート（分類 → 崩壊影響 → テスト+閾値 → Go/No-Go）

検証駆動設計の核。もっともらしい戦略を**反証可能な賭けの集合**に変え、最も危険な仮説から最も安く検証する。仮説表は `.spec/discovery/assumptions.md`（worksheet の仮説表をそのまま使う）。

## 全仮説を3分類する

- **Desirability（欲しいか）** — 問題は実在し上位にランクされるか。代替よりこの解決が選ばれるか
- **Viability（儲かるか）** — 支払意思・価格・CAC・チャネル・ユニットエコノミクス
- **Feasibility（作れるか）** — 技術・データ・法規制の主要リスク

## 崩壊影響でランク付け（Riskiest Assumption Test）

各仮説に問う: **「これが間違っていたら、戦略のどれだけが崩れるか?」** 最も危険で最も不確実な仮説を、**MVP より前に、最初に、最も安く**検証する。

## テストカタログ（仮説ごとに最も安い確からしいテストを選ぶ）

| テスト | 検証するもの | コスト |
|---|---|---|
| 顧客/乗り換えインタビュー | desirability・JTBD | 低 |
| スモークテスト / fake-door LP | 需要（登録率・クリック率） | 低 |
| コンシェルジュ MVP | desirability + 提供可能性 | 中 |
| Wizard-of-Oz | バックエンドなしで desirability | 中 |
| **プレセール / LOI（意向表明）** | **viability — 最強のシグナル** | 中 |
| Van Westendorp PSM（高すぎ/高い/安い/安すぎの4問） | 許容価格帯 | 低 |

## kill / pivot 閾値（上位仮説ごとに必須）

閾値は**テストを走らせる前に**決める。例: 「LP 登録率 < 5% なら需要仮説を棄却しスコープへ戻る」。**事前確約の閾値がないテストは検証ではない。**

## 価格は仮説であって算術ではない

`LTV:CAC ≥ 3` はスプレッドシートを検証するだけで市場を検証しない。ビジョン/スコープ由来の価格・CAC の仮置き値は、プレセールか Van Westendorp で検証する — 計算で正当化しない。

## Go / No-Go 判定

- **No-Go** — 崩壊クリティカルな仮説のどれかが「未検証」かつ「テスト+閾値も未定義」
- **Go** — それ以外。未解決の仮説は open リストとして追跡する

判定材料を人間に提示し、Discovery Gate の裁定（`sdd-core` スキルのゲート規約が正）を受ける。エビデンスが届いたらゲートは再訪する — 一度きりの儀式にしない。docs/ の該当ドラフトが影響を受けたら再伝播（sdd-core の lifecycle 再伝播プロトコル）に乗せる。
