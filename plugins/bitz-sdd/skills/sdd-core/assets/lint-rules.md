# spec-lint 規則
## EARS構文
- 受入基準は `WHEN … THEN … SHALL` 形式。1文に WHEN は1つ、SHALL は1つ
- 受動態・主語の省略を避け、「システムは」を明示する
## 禁止語（測定不能語辞書。数値・閾値に書き換えること）
高速に, 速やかに, 適切に, 十分な, 使いやすい, わかりやすい, 柔軟に, 可能な限り, なるべく, 安全に(閾値なし), 大量の, 少量の, 頻繁に
## 必須項目
- verification_method が空の要件は approved 不可
- benchmark / load-test の要件は本文に数値閾値必須
