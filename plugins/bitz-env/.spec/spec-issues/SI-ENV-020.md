- **矛盾/曖昧の内容**: SKILL.md 手順4は部分撤去時に「レジストリを残し、撤去済み
  項目を記録から消す」ことを求めるが、記録から消す際の書式（取り消し線の有無、
  日時併記、残置エントリの明示方法など）を規定していない。run12
  （TC-03 スキルあり、`evals/env-destroy/runs/12-skill-tc03/`）ではテスト実行者が
  可読性重視で「取り消し線＋撤去済み日付＋残置注記」という書式を独自に構成し、
  対応するアサーション（部分撤去のためレジストリを削除せず、撤去済み項目だけ
  記録から消している）自体は満たした
  （`runs/12-skill-tc03/sandbox/.claude/bitz-env.local.md` で確認）が、
  SKILL.md に規定が無いため実行者・実行間で表現がばらつく可能性がある。
  将来 env-doctor 等がレジストリを機械的に解析して残存物（例: 意図的に残された
  AGENTS.md のマーカー区間）を追跡する用途を想定すると、書式の不統一は解析を
  困難にする。
- **提案する修正**: SKILL.md 手順4に、撤去済みエントリ・残置エントリの記録方法を
  明記する（例: 各エントリに `status: removed|retained` とその日時を付記する、
  または統一されたマーカー表現を規定する）。一般化のポイントとして、TC-03固有の
  文言をハードコードするのではなく「部分撤去が発生した場合のレジストリエントリ
  状態表現」全般に適用されるルールとして規定すること。
- **影響推定**: SKILL.md 手順4の追記のみ。既存の完全撤去フロー（レジストリ自体の
  削除）には影響しない。env-doctor がレジストリをパースする実装を持つ場合、
  フォーマット変更に追従が必要になる可能性がある（現時点で env-doctor 側の
  レジストリパース実装の有無は本評価の範囲外のため未確認）。
