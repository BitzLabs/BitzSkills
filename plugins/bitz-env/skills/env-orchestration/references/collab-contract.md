# 協調アダプタ契約 v1（正）

bitz-env と連携する外部エージェント連携プラグイン（協調アダプタ）が満たすべき取り決め。
プラグイン同士は互いのファイルを参照できないため、連携は本契約（命名と役割）と
プロジェクト内レジストリ（`.claude/bitz-env.local.md`）による疎結合で行う。

bitz-env はアダプタが1つも無くても完全に機能する。アダプタは純粋な追加である。

## 1. 標準スキルセット

アダプタは以下のうち提供するものを、**この名前のスキル**として持つ
（スキル名による連携が両プラットフォーム共通の作法のため）:

| スキル名 | 役割 | 必須度 |
| --- | --- | --- |
| `delegate` | well-scoped なサブタスクの委譲実行 | 必須 |
| `review` | 現在の diff・設計へのクロスモデル検証（合議型のメンバー） | 推奨 |
| `status` | 実行中・完了ジョブの確認 | 任意 |

## 2. 能力宣言

アダプタの plugin マニフェスト `metadata` に `collab` セクションを持たせる:

```json
{
  "name": "bitz-collab-example",
  "version": "0.1.0",
  "metadata": {
    "collab": {
      "agent": "対象エージェント名（例: agy, codex）",
      "strengths": ["量産", "長文読解", "web検索"],
      "break-even": "委譲が割に合う目安の一文（例: 3ファイル以上の一括生成）"
    }
  }
}
```

マニフェストに書けない場合は、アダプタのルートに `collab.json`（同じ形）を置いてもよい。
env-register はどちらも読む。

## 3. 報告形式（DIGEST）

- `delegate` は成果物全文ではなく **DIGEST（要約報告）** を返す:
  何を作った/変えたか、対象ファイル一覧、実行した検証とその実出力、未解決事項
- 「成功しました」だけの報告は契約違反。検証の実出力を必ず含める
- 検収（機械チェックの再実行）は呼び出し元（中心）の義務であり、
  アダプタは検収を代行しない

## 4. 互換性

- 既存の antigravity プラグイン（delegate / review / status 構成）は
  本契約の事実上の先行実装であり、v1 はその一般化である
- 契約の拡張は後方互換で行う（既存フィールドの意味を変えない）
