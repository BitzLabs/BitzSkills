# bitz-env — 開発環境展開プラグイン

培った開発環境 — **ガードレール**（危険操作の機械ブロック）と**モデル非依存の協調運用**
（委譲型・相談型・合議型）— を、任意のプロジェクトへ展開するプラグインです。

> **棲み分け**: `plugin-creator` は「プラグインの作り方」、`skill-creator` は「スキルの作り方」、
> 本プラグインは「開発環境そのものの展開」を担当します。

## 2層構造

| 層 | 実体 | 効き始め |
|---|---|---|
| **即効層（同梱フック）** | PreToolUse フックが `rm -rf` / `git push --force` / `git reset --hard` / `git clean -f` / `sudo` を機械ブロック。あわせて `rules/*.md` のガードレール文書を Antigravity はネイティブ rules として、Claude Code は SessionStart フック注入としてコンテキストへ読み込む（注入対象はガードレール本文に限定し、肥大化を避ける。過大になったら要約・分割を検討する） | プラグイン有効化直後から。プロジェクトに何も書き込まない |
| **生成層（env-init）** | permissions（settings.json）・AGENTS.md 雛形・CLAUDE.md 断片・advisor/worker サブエージェントをユーザー確認付きで生成 | env-init 実行後。プラグイン無効時も効き続ける恒久層 |

同じガードレールを2層で持つのは意図的な二重化です（片方が無効でももう片方が守る）。
3層（permissions ⇔ フック ⇔ ナラティブ）の同期ズレは `env-doctor` が検出します。

同梱フックは「誤操作の抑止」であり、悪意ある回避を防ぐセキュリティ境界ではありません。
正規表現ベースの検出はコマンド置換・エンコード・環境変数展開などで回避可能です。
恒久的な防御は env-init が生成する permissions 層が担います。

## モデル非依存の協調運用

司令塔（中心）は**ユーザーがセッションで選択しているモデル**。特定モデルを前提にしません。
役割は中心からの相対位置で定義します（advisor = 上位への相談先 / worker = 下位への委譲先）。

| パターン | 向き | 使いどころ |
|---|---|---|
| 委譲型 | 中心 → worker | 定型作業。トークン温存 |
| 相談型 | 中心 → advisor | 難所の判断材料を仰ぐ。最終判断は中心 |
| 合議型 | 中心 → 複数へ並列諮問 | 高影響の判断のみ。裁定は多数決でなく中心 |

advisor が使えない環境では相談をスキップして劣化動作します（環境全体は止まりません）。

## 収録スキル

| スキル | 役割 |
|---|---|
| `env-init` | 環境の対話的展開（permissions / AGENTS.md / CLAUDE.md 断片 / advisor・worker 生成。モデルはセットアップ時に選択） |
| `env-orchestration` | 協調パターンの選択と実行の案内（決定木・検収義務・合議の進め方） |
| `env-register` | 協調アダプタ（外部エージェント連携プラグイン）の検出・レジストリ登録・委譲マトリクス更新 |
| `env-doctor` | ガードレール3層と協調構成の同期ズレ診断 |
| `env-destroy` | env-init 生成物のトラッキング記録に基づくユーザー確認付き撤去（マーカー区間のみ除去） |

## 協調アダプタ

外部エージェント（Gemini / Codex 等）との連携は、契約ベースの疎結合で後付けできます
（契約: `skills/env-orchestration/references/collab-contract.md`、
レジストリ: 展開先の `.claude/bitz-env.local.md`）。
アダプタが1つも無くても本プラグインは完全に機能します。

## インストール

```
# Claude Code
/plugin marketplace add <このリポジトリ>
/plugin install bitz-env@bitzskills

# Antigravity 2.0
agy plugin install <リポジトリ>/plugins/bitz-env
```

導入後、対象プロジェクトで「開発環境を展開して」（env-init）から始めてください。

## 注意事項

- このプラグインは以下の2つのフックを含み、有効化により自動で動作します。
  Claude Code がインストール時に同意を求める場合はその場で判断でき、求めない場合も
  下記のとおり動作範囲は限定的です:
  - **PreToolUse フック**: 破壊的操作（rm -rf / git push --force / git reset --hard /
    git clean -f / sudo）を**ブロックするだけ**。ユーザーに代わる実行や外部送信は行いません
  - **SessionStart フック**: `rules/*.md` のガードレール文書を**コンテキストへ注入するだけ**。
    副作用はありません
- env-init の生成物はプラグイン更新に自動追従しません。`env-doctor` で差分を確認してください
