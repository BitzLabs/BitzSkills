# 標準ライフサイクルスキル（init / doctor / update / uninstall）

全プラグインに共通するライフサイクル操作は、独自命名を乱立させず標準スキル名
`<plugin名>:init` / `doctor` / `update` / `uninstall` として提供する（CORE-CON-008）。
この4操作に該当しない独自スキルは、従来どおりプラグイン固有の命名でよい。

## 最小契約

| スキル | 責務 | 書き込み | 典型的な確認観点 |
|---|---|---|---|
| `init` | 初期設定と依存確認。対象プロジェクトへの初回展開 | 対象プロジェクトへの生成物配置（事前確認事項に該当する場合あり） | 依存プラグイン・前提条件の充足確認、二重初期化の防止 |
| `doctor` | 環境診断。展開済み状態の健全性チェック | 読み取り専用 | 期待される生成物・設定が揃っているか、他コンポーネントとの整合 |
| `update` | バージョン更新と依存再確認 | 差分のある生成物のみ | 現在バージョンと最新バージョンの差分検出、依存関係の再確認 |
| `uninstall` | 痕跡を残さない撤去 | 自身が生成したものの削除のみ（他の生成物は対象外） | 撤去後に生成物が残っていないこと、ユーザーの明示確認を経ること |

## 命名の運用

- 標準4操作すべてを備える必要はない。提供する操作だけを標準名で命名する。
- `uninstall` は撤去という破壊的操作を含むため、AGENTS.md のガードレール
  （リポジトリ外への書き込み・削除は事前確認）に従い、実行前にユーザー確認を取る。
- 先行実装が独自名を使っている場合は、標準への追随（改名）を検討する。
  改名は当該プラグインのライフサイクルタスクとして実施し、旧名への参照が
  残らないことを確認する（例: bitz-env の `env-destroy` → `env-uninstall`）。
