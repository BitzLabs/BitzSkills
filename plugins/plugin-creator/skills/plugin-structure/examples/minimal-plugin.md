# 最小プラグインの例

コマンド1つだけの最小構成プラグイン。

## ディレクトリ構造

```
hello-world/
├── .claude-plugin/
│   └── plugin.json
└── commands/
    └── hello.md
```

## ファイル内容

### .claude-plugin/plugin.json

```json
{
  "name": "hello-world"
}
```

### commands/hello.md

```markdown
---
name: hello
description: フレンドリーな挨拶メッセージを表示する
---

# hello コマンド

ユーザーにフレンドリーな挨拶を表示する。

## 実装

以下のメッセージをユーザーに出力する:

> こんにちは！hello-world プラグインのシンプルなコマンドです。
>
> より複雑なプラグインを作る出発点として使ってください。

コマンドが正常に実行されたことを示すため、現在時刻も挨拶に含める。
```

## 使い方

プラグインをインストールした後:

```
$ claude
> /hello
こんにちは！hello-world プラグインのシンプルなコマンドです。

より複雑なプラグインを作る出発点として使ってください。

実行時刻: 2026-07-05 14:30:22 JST
```

## ポイント

1. **最小マニフェスト**: 必須の `name` フィールドのみ
2. **単一コマンド**: `commands/` にマークダウンファイル1つ
3. **自動発見**: Claude Code がコマンドを自動で見つける
4. **依存なし**: スクリプト・フック・外部リソースなし

## このパターンが向く場面

- 短時間のプロトタイプ
- 単機能ユーティリティ
- プラグイン開発の学習
- 特定機能1つだけの社内ツール

## 拡張するには

1. **コマンド追加**: `commands/` に `.md` ファイルを増やす
2. **メタデータ追加**: `plugin.json` に version / description / author を追加
3. **エージェント追加**: `agents/` フォルダを作成
4. **フック追加**: `hooks/hooks.json` を作成
