## PoP / PoD

### PoP（同等性のポイント — 満たさないと土俵に乗れない）

- プラグインの基本ディレクトリ構造・plugin.json・自動発見を正しく説明できる
- コマンド / エージェント / フック / MCP / 設定の基本的な書き方を提示できる
- （これらは公式ドキュメントも満たす。**超過しても勝てないので過剰投資しない**）

### PoD（差別化のポイント — 資源を集中する）

1. **両プラットフォーム仕様差の一次情報化**: Claude Code と Antigravity 2.0 の差分（コマンドのスキル変換・2マニフェスト・フック機構・MCP 設定形式）を明示的に織り込む。実測ベース（Gemini 生成解説を根拠にしない規律）
2. **コンポーネント別 progressive disclosure**: 7スキルが必要な時だけ発動し、context を薄く保つ
3. **設計 → 検証の一気通貫**: create-plugin コマンド + plugin-validator エージェントで「発見されず動かない」を出荷前に潰す
4. **責務分離の明確さ**: スキル本体は skill-creator へ誘導し、重複や混乱を生まない

> PoD 1・4 は反証可能な差別化仮説でもある。崩壊クリティカルなものは assumptions.md（PLG-DSC-006）へ転記する。

## ポジショニング・キャンバス（April Dunford）

1. **競合代替**: 分散した公式ドキュメント / 断片的な解説記事 / 既存プラグインのコピー / 手書き
2. **独自の属性**: 両プラットフォーム仕様差の一次情報を、コンポーネント別に必要時だけ提示する構造 + 出荷前の機械検証
3. **価値（成果）**: 「調べ直しなしで、両対応で発見・動作するプラグインを、配置ミスなく出荷できる」
4. **ターゲットセグメント**: 両対応プラグインを作る Claude Code / Antigravity 開発者（主要 = BitzSkills 内製メンテナ）
5. **市場カテゴリ**: プラグイン開発支援ツール（両対応特化）

## ポジショニング・ステートメント

> **両対応プラグインを作る Claude Code / Antigravity 2.0 開発者** のための **plugin-creator** は、**プラットフォーム仕様差に迷わず正しい雛形を最短で作りたいというニーズ** に対する **プラグイン開発支援ツール** であり、**コンポーネント別ガイド + 出荷前の機械検証で「発見されず動かない」を防ぐこと** を提供する。**分散した公式ドキュメントや断片的な解説記事** と違い、**両プラットフォームの仕様差を一次情報として体系化し、必要な時だけ提示する（PoD）**。

## 規律

- 主張するのは**価値（顧客の成果）**であって機能リストではない
- PoP（基本的な書き方の網羅）で公式ドキュメントと消耗戦をしない。資源は PoD（両対応の一次情報化・責務分離）に集中する
