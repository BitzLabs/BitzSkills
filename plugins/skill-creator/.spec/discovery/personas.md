## Jobs-to-be-Done（ジョブストーリー形式）

- **J1（作成）**: 新しい能力をエージェントに教えたいとき、仕様に沿った SKILL.md 一式を
  手早く組み立てたい。作法で迷わず、後戻りしないように。
  - 機能的: 準拠した雛形を得る／感情的: 「これで合っているはず」と安心したい／
    社会的: OSS として出しても恥ずかしくない品質に見せたい。
- **J2（検証）**: スキルを書き終えたとき、Agent Skills 仕様に本当に準拠しているか
  機械的に確かめたい。自己申告ではなくチェックリストで。
- **J3（効果測定）**: そのスキルが「効いているのか」を、スキルあり/なしの比較で
  客観的に知りたい。体感で良し悪しを決めたくない。
- **J4（配置）**: 完成したスキルを Claude Code と Antigravity の双方へ、パスや手順で
  悩まず配りたい。バージョン更新も安全に。
- **J5（継続改善）**: 配置後にスキルが空振り・劣化したとき、勘ではなく蓄積した
  観察ログを起点に、次の改善を人間の裁定に載せたい。

ジョブはソリューション非依存で安定。ペルソナをジョブに繋留する。

## ペルソナカード

### ペルソナ A: 「一人パイプライン」開発者 hide（主要ペルソナ）

- **アーキタイプ／1行**: BitzSkills モノレポでスキルを量産・保守する個人開発者。
- **文脈と行動**: Claude Code を司令塔に、Antigravity も併用。1 人で作成から配置・
  改善まで全工程を回す。トークン消費と保守工数に敏感で、機械検証を信頼する。
- **Jobs**: J1〜J5 のすべて（作成→検証→効果測定→配置→継続改善を通しで担う）。
- **Pains**: 作法が暗黙知でブレる／効果が体感頼み／配置先ごとの手順差／
  配置後の劣化に気づけない。
- **Gains**: 一貫した作法・あり/なし比較で可視化された効果・ワンストップ配置・
  観察ログ起点の改善サイクル。
- **生の声**: 「作ったスキルが本当に効いているのか、勘で判断したくない」
  [proto / 未検証]（本人の設計意図からの再構成。インタビュー記録ではない）。

### ペルソナ B: OSS スキル作者（二次ペルソナ）

- **アーキタイプ／1行**: Agent Skills 標準でスキルを作りたいが作法・検証・配置で
  迷う外部開発者。本プラグインを `/plugin install` で導入する層。
- **文脈と行動**: スキルは書けるが、frontmatter 仕様・description の発動精度・
  progressive disclosure といった作法に確信が持てない。
- **Jobs**: 主に J1・J2・J4（作成・検証・配置）。J3・J5 は使いこなせば到達。
- **Pains**: 何が「正しい SKILL.md」か分からない／自分のスキルが発動しない原因が不明。
- **Gains**: 準拠テンプレと検証チェックリストで「これで合っている」という確信。
- **生の声**: `TBD`（外部ユーザーは未接触。実在需要も未検証）。

### ペルソナ C（served / anti-persona）

- **served**: スキルを消費するだけのエージェント利用者 — 本プラグインの成果物の
  恩恵は受けるが、直接のユーザーではない（最適化対象にしない）。
- **anti-persona**: プラグイン構造・SDD ワークフロー・DDD を本プラグインに求める人 —
  それぞれ plugin-creator / bitz-sdd / bitz-ddd の担当（scope.md の Won't 参照）。

## 主要ペルソナの特定

**主要ペルソナ = A「一人パイプライン」開発者 hide**。プロダクトはまずこの 1 人が
全工程を最小摩擦で回せることに最適化する。B（OSS 作者）は secondary で、A 向けに
磨いた作法・検証がそのまま外部にも効く、という順序で価値を波及させる。

## 機会リスト（優先順・ジョブ紐づけ）

1. **効果測定の摩擦低減**（J3 / ペルソナ A）— あり/なし比較を回す手間が最も
   レバレッジが高い（NSM I2 に直結）。感情曲線が最も沈むのは「効いているか分からない」瞬間。
2. **配置後の劣化検知**（J5 / ペルソナ A）— observer 計装の被覆を上げ、空振りを拾う。
3. **作法の確信提供**（J1・J2 / ペルソナ B）— 外部採用の入り口。
   ※ 上記はいずれもペルソナ／ジョブに紐づく。生の声・感情曲線の裏づけは未収集のため
   `TBD`（ユーザーリサーチで昇格させる）。
