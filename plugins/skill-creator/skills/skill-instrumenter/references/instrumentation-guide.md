# 計装ガイド

`skill-instrumenter` が SKILL.md に注入する自己観察セクションの正式な
テンプレートと運用ルール。

## 注入テンプレート（この文言をそのまま使う）

対象 SKILL.md 本文の末尾に、空行を1行挟んで以下を追記する。

```markdown
<!-- bitzskills-observer:start v1 -->
## 自己観察（skill-observer）

このスキルの作業を終えたら（完了報告の後に）、`skill-observer` を使って
今回の実行を自己観察する。問題（partial / fail）があった場合のみ観察ログに
記録し、問題がなければ何も記録しない。観察・記録の失敗によって、このスキル
本来の作業結果を損なわないこと。
<!-- bitzskills-observer:end -->
```

## マーカーの規約

- 開始: `<!-- bitzskills-observer:start v<N> -->`（`v<N>` はテンプレート版数。
  現行は `v1`）
- 終了: `<!-- bitzskills-observer:end -->`
- 検出・更新・除去はすべてこのマーカーを基準に行う。マーカーを手で編集しない
- テンプレートを改訂したら版数を上げ、このファイルの履歴に追記する

## 冪等性ルール

| 対象の状態 | 操作 |
| --- | --- |
| マーカーなし | 末尾にテンプレートを追記 |
| マーカーあり・同一版数 | 何もしない（計装済みと報告） |
| マーカーあり・旧版数 | マーカー間を現行テンプレートで置換 |

いずれの場合も、変更が発生したときだけ patch bump する。

## 複数スキルの一括計装

複数スキルを計装する場合も1スキルずつ上の手順を適用し、最後にまとめて
報告する。途中で失敗したスキルがあっても他のスキルの計装は継続し、
失敗分を報告に明記する。

## 計装対象の選び方（目安）

- 実運用で使用頻度が高いワークフロースキル → 計装する価値が高い
- 単発の実験用・使い捨てスキル → 計装不要
- `skill-observer` / `skill-improver` 自身 → 計装しない（再帰を避ける）
