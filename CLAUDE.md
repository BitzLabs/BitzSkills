# CLAUDE.md

共通ルール（役割・構成・ガードレール・規約・手順）はすべて @AGENTS.md を正とする。

## Claude Code 固有

- スキル開発の全工程は `skill-pipeline` スキルが統括する
  （creator → validator → tester → evaluator →不合格なら optimizer で反復→ packager。
  配置後は instrumenter / observer / improver の自己改善ループ）
- version bump は `/bump <plugin名> [major|minor|patch]`、
  プラグイン追加は `/add-plugin <plugin名>` を使う（`.claude/commands/` のリポジトリ専用コマンド）
- 危険操作の機械的ブロックは `.claude/settings.json` の permissions で強制している
  （AGENTS.md のガードレール節と対応。緩めるときは両方を見直す）

## オーケストレーター運用（最上位モデル = 司令塔）

最も賢いモデルは作業者ではなく司令塔にする。司令塔は**計画立案・タスク分解・
委譲先の選定・結果の検収・統合・最終判断**に徹し、実作業は下位モデルの
サブエージェント／antigravity（Gemini）へ委譲してトークンを温存する。
委譲は**役割**で判定し、具体モデルは下の**委譲レジストリ**に集約する
（モデル世代交代時はレジストリと `.claude/agents/*.md` の局所更新で追随する。SI-CORE-021 / DSN-001）。

### 委譲レジストリ（機械検証対象 = SSOT）

**ティアはしご**（上位→下位）: `Fable 5 > Opus > Sonnet > Haiku`

| 役割 | 委譲先 | ティア |
|---|---|---|
| 深い推論・設計案作成・複雑な不具合調査・トレードオフ分析 | `deep-reasoner` サブエージェント | Opus |
| 定型修正・テスト実行・ファイル整理・frontmatter 一括更新 | `fast-worker` サブエージェント | Sonnet |
| コードベースの探索・広範な検索 | 組み込み `Explore` エージェント | — |
| 量産系（スキャフォールド・網羅テスト生成・一括移行・長文読解の要約・Web検索） | `antigravity-delegate` / `/antigravity:delegate` | 別PF |
| セカンドオピニオン・クロスモデル検証 | `/antigravity:review` | 別PF |

- この表が Claude 側の委譲判断の唯一の正（SSOT）。委譲先のティアは `.claude/agents/*.md` の
  `model:` と一致させる。`release_check.py` が整合を機械検証する（委譲先 agent の実在・
  ティアはしごとの整合・モデル名のルーティング文書への直書き禁止）。
- Antigravity（Gemini）のモデルルーティングは antigravity プラグインが別途所有し、本表とは**分離管理**。
- Codex（OpenAI プラグイン）は**使用しない**。別視点の検証は antigravity（Gemini）で行う。

### 相対選択（司令塔ティア基準）

委譲・相談は司令塔（＝ユーザが起動したモデル）のティアはしご上の**相対位置**で決まる:

- **下位への委託**: 機械的・量産タスクは、役割の委譲先ティアが司令塔より**下位**の時のみ委譲する。
  下位が無ければ司令塔が自己実行する。役割の束縛ティア == 司令塔ティアなら省トークンにならないため**無効化**。
- **上位への相談・上申**: 難調査・設計タスクは、司令塔より**上位**ティアが存在する時のみ相談・上申する。
  司令塔が最上位なら相談相手なし＝自己判断する。
- **フォールバック**: Fable 5 が使えず Opus が司令塔でも、司令塔は作業者にならない。機械的作業は
  Sonnet（fast-worker）へ委譲する（deep-reasoner=Opus は同一ティアで無効）。
- antigravity（Gemini）は別プラットフォームで相対階層と**直交**。量産・クロス検証はいつでも可。

SDD の Execute フェーズではこの判定を `sdd-implement` の委譲ゲート（`delegation-routing.md`）が実行する。

### 運用ルール

- **委譲の損益分岐**: 小さな単発タスク（1ファイルの軽微な編集等）は委譲せず司令塔が直接行う。
  委譲の往復コストが節約分を上回るため
- **コンテキストを薄く保つ**: 委譲先が生成したファイル全文を読み戻さない。
  要約（DIGEST）だけ受け取り、検証は release_check / validator 等の機械チェックで行う
- **返答形式を明示**: 委譲時は期待する成果物・報告形式・検証方法を指示に含める
- **検収は司令塔**: 委譲先の「成功しました」を信用せず、AGENTS.md の検証義務に従い
  司令塔が自分で（またはスクリプトで）確認する
- **並列は 5〜6 体まで**: サブエージェントの同時起動はこれを上限とする
- **フォールバック**: Fable 5 が使えない場合は Opus を司令塔として同じ運用を継続する
